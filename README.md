# Amaely website (light, split layout)

A single page on a soft white background. The screen is split. The top half is frozen:
the morphing 3D art sits there, with the brand "Amaely" and the line "A living universe
of care" pinned on top of it, always visible. The bottom half scrolls. As you scroll,
the art quietly reshapes (mind, heart, nourishment, strength, rest, protection, the
whole self, and back to the wider universe) while short caring lines flow past one at a
time, fading gently in and out so there are no hard section breaks. The scroll is short
and brisk. The message is simple: whole person healthcare for everyone, with parents and
elders always first. No pricing, no jargon.

Everything is in ONE file: `index.html`. The 3D engine is built in, so it works offline
and on any host. No build step, no server needed.

## IMPORTANT: how to view it properly
The chat preview window often cannot run 3D graphics or load web fonts, so it may look
flat or plain there. To see the real thing:
- Download `index.html`, then double click it to open in Chrome, Safari or Edge, OR
- Put it online with the free steps below.
On a real browser the soft particle field drifts and reshapes as you scroll.

## It is built for every screen
Phone, tablet, laptop and desktop are all handled. Type, spacing and the 3D scene resize
to the screen, particle counts drop on phones to stay smooth, and anyone who turns on
"reduce motion" gets a calm, still version that is still fully readable.

## Put it online free (GitHub Pages), about 5 minutes
1. Create a free account at github.com.
2. Make a new repository (for example `amaely-site`), set it to Public.
3. Upload `index.html`, `logo.svg`, `logo-mark.svg`, `ig-post.png`, `ig-profile.png`.
4. Open Settings, then Pages. Under "Build and deployment", set Source to "Deploy from a
   branch", branch `main`, folder `/ (root)`, then Save.
5. Wait about a minute. Your site is live at `https://YOURNAME.github.io/amaely-site/`.

### Using your own domain
1. In the same Pages settings, type your domain in "Custom domain" and Save.
2. At your domain registrar, add the DNS records GitHub shows you (four A records, or a
   CNAME for a `www` address).
3. Tick "Enforce HTTPS" once it becomes available.

Note on the name: the brand on the site is spelled "Amaely", but the domain you own is
"amaley.com" (the e and l are swapped). The buttons and footer currently point to
amaley.com so they match the domain you own. You will want to either buy amaely.com, or
change the brand everywhere to "Amaley". This is still an open decision.
